8. OpenGL3DVision (Helifax)
============================================================================
OpenGL 3D Vision wrapper — Windows opengl32.dll proxy that adds NVIDIA 3D
Vision support to OpenGL games via shader injection and FBO-based per-eye
rendering.
https://github.com/Helifax/OpenGL3DVision

Used in:
  (a) NvDirectMode/d3d9, /d3d10, /d3d11 magic-header SBS capture path
      (NvDirectMode/{d3d9,d3d10,d3d11}/magic_header_capture.{h,cpp})
  (b) NvDirectMode/opengl32 (source only — not shipped in v0.2.x releases)

(a) Magic-header SBS capture: the NVSTEREOIMAGEHEADER struct + signature
constant (0x4433564E "NV3D") + flag bits (SIH_SWAP_EYES / SIH_SCALE_TO_FIT)
+ the documented surface convention (W*2 × H+1 stereo blit with the header
in the last row) come from OpenGL3DVision's src/src/opengl_3dv.cpp. WiZ3D
implements the *consumer* side of that protocol — detecting these surfaces
when games (or wrappers like OpenGL3DVision itself) submit them, and
splitting them per-eye for our own composite pipeline. This is the
documented NVIDIA 3D Vision SDK "Stereo Blit Defaults" technique; the
OpenGL3DVision source is the clearest publicly-available reference for it.

(b) NvDirectMode opengl32 (legacy): the previous OpenGL Direct Mode wrapper
adapted OpenGL3DVision's two-FBO per-eye design (one FBO per eye sharing a
depth/stencil renderbuffer) and its wglGetProcAddress-based hook strategy
for GL extension functions. As of v0.2.1 that opengl32 build is no longer
shipped — magic-header capture (a) on the DX9/10/11 side picks up
OpenGL3DVision-driven games via its OpenGL↔DX interop bridge automatically.
The opengl32 source is kept in the tree (NvDirectMode/opengl32/) but not
deployed.

Neither path redistributes or uses OpenGL3DVision's shader injection,
FPS injection, or 3D Vision profile management code.
